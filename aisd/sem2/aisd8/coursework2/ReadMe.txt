Набор учебных материалов к курсовой работе (STL): измерение временной сложности алгоритма в эксперименте на ЭВМ 

1. in.txt	- образец данных - результатов эксперимента (набор пар <размер входа, время>); 
2. RG41.exe	- программа для статистической обработки результатов (регрессионный анализ);
3. out.txt	- результат обработки файла in.txt программой RG41;
4. EXAMPLE.xls	- электронная таблица с заготовкой для сводного анализа файлов in.txt и out.txt;

Порядок выполнения статистического эксперимента.
1. Получить статистику в виде текстового файла in.txt. Файл может называться иначе, но он должен быть текстовым.
В нём - любое количество, но не менее 10 пар чисел
размер-входа время
Размер входа - целое число, а время считается типа double (хотя может выглядеть как целое, т.е. без точки) - см. прилагаемый пример.
Количество данных указывать не нужно, пр-ма сама подсчитает.
ВНИМАНИЕ: В отсчётах времени для отделения дробной части должна использоваться десятичная точка, использование запятой не допускается. Файл с запятыми следует отредактировать, заменив запятые точками.

2. Поместить файл in.txt в один каталог с программой RG41 и запустить её. 
Пр-ма создаст файл out.txt, выдав при этом оба файла на экран для просмотра на случай сбоев.

3. Файлы in.txt и out.txt поместить в один каталог с заготовкой example.xls
Импортировать их в таблицу на свободное место и скопировать в нужное согласно инструкции в ней.

4. Проанализировать вспомогательную табличку отношений дисперсий и отобрать наиболее подходящую модель регрессии по критерию Фишера.

5. Выбрать соответствующий модели график для помещения в отчёт по КР. 
Перед использованием графика проверить и подкорректировать, если нужно, границы диапазонов всех его исходных данных
в соответствии с реальным к-вом измерений. 
Первые две колонки упорядочить по возрастанию мощности.
Количество строк в колонках C - AY должно соответствовать к-ву пар измерений в колонках A и B.
Если строк недостаточно - размножить их копированием существующих; настроить график на фактическое к-во строк таблицы.

6.Программа RG41 отличается от предыдущих версий (RG32 и т.п.) наличием дополнительной опции.
В последней колонке выходного файла out.txt имеются пометки - результаты сравнения текущего уравнения с последующими по критерию Фишера.
Пометок две - для допустимой ошибки 5% и 1%.
"+" означает, что текущее уравнение целесообразно заменить следующим;
"-" означает, что уравнение окончательное, и усложнения далее не значимы;
"*" означает, что данное уравнение следует отвергнуть как не имеющее физического смысла (отрицательный старший к-т).

7. Программа RG41 принимает дополнительный числовой аргумент, предшествующий (не обязательному) имени входного файла.
Если аргумент - число от 1 до 9, выдача в файл out.txt прекращается по выдаче итогового уравнения (помеченного "--").
 Если число - 10 и более, программа делит данные на порции указанного размера и обрабатывает их отдельно (последняя порция отсчитывается с конца, с возможным перекрытием).
Так можно заметить возможный тренд получаемых уравнений регрессии.
Для такой обработки исходные данные рекомендуется упорядочить по содержимому первого столбца. 
Обработка данных сразу целиком этого не требует. Упорядочение понадобится только после ввода данных в Таблицу для получения корректных графиков.
